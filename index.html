LeetCode 461 汉明距离 - 三种解法从暴力到位运算

Posted on 2026-05-10

前两天刷题遇到 LeetCode 461 汉明距离，题目看着挺简单的，但仔细想了想发现可以从好几个角度去解决，顺便把二进制中 1 的计数问题也梳理了一下。

题目

给定两个整数 x 和 y，计算它们之间的汉明距离。汉明距离就是两个数字对应二进制位不同的位置的数量。

比如 x = 1, y = 4：

第0位和第2位不同，所以汉明距离是 2。

思路

说白了，汉明距离就是看两个数异或之后结果里有多少个 1。因为异或的规则是相同为 0，不同为 1，所以异或结果中 1 的个数恰好就是不同位的个数。

那问题就转化成了：怎么数一个整数二进制表示中 1 的个数？

解法一：逐位判断

最直觉的想法，一位一位看就行了。用一个掩码从最低位开始，每次左移一位，逐个检查。

这个写法没什么花哨的，就是老老实实每一位都看一遍。时间复杂度 O(32) 也就是 O(1)，空间 O(1)。但缺点是哪怕 z 只有1位是 1，也要跑满 32 次循环，有点浪费。

稍微优化一下，可以改成右移 z 的方式，这样循环次数还是 32，但写法更简洁：

注意这里用 >>> 而不是 >>，因为 >> 是带符号右移，高位会补 1，可能导致死循环。>>> 是无符号右移，高位补 0，这才是我们想要的。

解法二：n & (n-1) 技巧

这个是我觉得最值得记住的一个位运算技巧。n & (n-1) 的效果是把 n 的最低位的 1 变成 0。

拿个例子试试，n = 12，二进制是 1100：

每做一次就消掉一个 1，那执行了几次就有几个 1。这样循环次数就等于 1 的个数，比逐位判断要快不少。

这个写法的时间复杂度取决于 1 的个数，最坏情况 O(32)，平均情况要好很多。

说实话第一次看到 n & (n-1) 这个操作的时候愣了一下，自己拿笔算了几个数才理解。原理其实也好懂：n-1 会让最低位的 1 变成 0，同时它右边的 0 全变成 1，所以与运算之后，最低位的 1 及其右边的位全变 0，左边的位不受影响。

解法三：库函数

Java 里 Integer.bitCount() 可以直接返回二进制中 1 的个数，底层也是高效实现。

一行就完事了。面试的时候如果先写了解法二再提一下这个库函数，我觉得是比较加分的，说明你知道底层原理，也知道工程上怎么偷懒。

另外 Java 里还有个不太常见但也能用的方式，Integer.toBinaryString(x ^ y) 转成字符串然后数 1，代码就不贴了，性能一般，知道就行。

小结

三种方法放在一起对比一下：

解法 时间复杂度 特点
逐位判断 O(32) 最直观，但不管1多1少都要跑满
n & (n-1) O(1的个数) 经典位运算技巧，面试常考
Integer.bitCount O(1) 工程首选，但要了解原理

这题本身不难，但 n & (n-1) 这个技巧后面还会经常遇到，比如判断一个数是不是 2 的幂（2 的幂只有一个 1，做一次 n & (n-1) 就变 0 了），建议把这个操作记熟。
